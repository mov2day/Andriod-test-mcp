Technical Specification

A Python-based Model Context Protocol server that enforces configurable QA automation strategies — enabling AI agents to analyse repositories, generate prescriptive test plans, validate test output, and enforce quality gates through a plug-and-play strategy plugin architecture.

01System Overview

QE-MCP is an MCP server written in Python that acts as a QA strategy enforcement layer for AI coding agents. It does not call LLMs itself. It loads a strategy, analyses source code deterministically, assembles a prescriptive test plan, and returns structured briefs that the calling agent uses to generate tests.

The central design insight: strategy and enforcement are separate concerns from generation. The MCP owns the rules, the structure, and the quality gate. The AI agent owns the generation. This separation prevents strategy drift and makes every generated test traceable to a declared rule.

What QE-MCP Does
  • Loads and validates strategy plugins at runtime
  • Discovers source files and classifies them by type
  • Produces prescriptive TC-001..N test plans
  • Validates generated tests against declared strategy rules
  • Enforces coverage thresholds via a hard gate tool
  • Emits structured JSON reports and structured logs
What QE-MCP Does NOT Do
  • Call any LLM or AI API directly
  • Execute test suites or run a test runner
  • Parse import graphs or build dependency trees (V1)
  • Perform semantic understanding of business logic
  • Store state between MCP server restarts
  • Replace human spec judgement on expected behavior

02Design Principles

Principle 1 — Strategy Owns the Rules
  • No QA logic is hardcoded in the MCP server core
  • Every classification, spec rule, and naming convention lives in the strategy plugin
  • Swapping strategy = swapping all QA knowledge
Principle 2 — Fail Loudly on Bad Contract
  • A strategy missing any abstract method raises StrategyLoadError at load time
  • A tool receiving an invalid strategy ID returns a structured error, never a silent no-op
  • All violations have a severity: error blocks; warning is advisory
Principle 3 — Deterministic Analysis
  • Repo analysis uses only Python ast and pathlib — no LLM calls
  • Same repo + same strategy = same output, always
  • classify_source_file() must be pure and synchronous; enforced via timeout wrapper
Principle 4 — Structured Over Narrative
  • Every tool returns typed, structured JSON — not prose
  • Reports have a defined schema enforced by Pydantic
  • Logs use structured format: JSON in CI, readable in dev

04Strategy Contract

The BaseStrategy abstract class is the single contract every plugin must satisfy. The registry validates it at load time using Python's abc module. A plugin that does not implement every abstract method cannot be registered — the server raises StrategyLoadError before any tool call is processed.

# strategy/base.py
from abc import ABC, abstractmethod
from dataclasses import dataclass, field
from typing import List, Optional, Dict, Any
from enum import Enum


class TestLaneType(Enum):
    UNIT        = "unit"
    INTEGRATION = "integration"
    CONTRACT    = "contract"
    E2E         = "e2e"
    PERFORMANCE = "performance"


@dataclass
class TestLane:
    lane_type:          TestLaneType
    required:           bool
    file_glob:          str          # e.g. "tests/unit/**/test_*.py"
    coverage_threshold: float        # 0.0–1.0; 0.0 = not measured
    max_allowed_skips:  int = 0


@dataclass
class NamingConvention:
    test_file_pattern:   str  # regex, e.g. r"test_(?P<source>.+)\.py"
    test_method_pattern: str  # e.g. r"test_(?P<action>.+)_(?P<condition>.+)_(?P<expected>.+)"


@dataclass
class BehavioralSpec:
    """
    Defines what a valid test spec looks like for a given source class type.
    Without this, generated tests mirror implementation bugs.
    """
    source_class_type:      str
    required_spec_fields: List[str]   # ["given", "when", "then", "expected_state"]
    required_assertions_min: int = 1
    mutation_sensitive:     bool = False


@dataclass
class SourceClassification:
    class_type:         str                  # "service" | "utility" | "dto" | ...
    complexity_score:   int                  # 1–10
    testability_issues: List[str]            # ["hidden dependency", ...]
    required_lanes:     List[TestLaneType]
    ast_metrics:        Dict[str, Any] = field(default_factory=dict)
    # e.g. {"functions": 12, "classes": 3, "cyclomatic_complexity": 7}


class BaseStrategy(ABC):

    @property
    @abstractmethod
    def id(self) -> str:
        """Unique strategy identifier. Used in all tool calls and report headers."""

    @property
    @abstractmethod
    def name(self) -> str: ...

    @property
    @abstractmethod
    def version(self) -> str: ...

    @property
    @abstractmethod
    def target_language(self) -> str: ...   # "python" | "kotlin" | "typescript"

    @property
    @abstractmethod
    def target_framework(self) -> Optional[str]: ...  # nullable

    @abstractmethod
    def get_test_lanes(self) -> List[TestLane]: ...

    @abstractmethod
    def get_naming_conventions(self) -> NamingConvention: ...

    @abstractmethod
    def get_behavioral_specs(self) -> List[BehavioralSpec]: ...

    @abstractmethod
    def classify_source_file(
        self,
        file_path: str,
        content:   str,
    ) -> SourceClassification:
        """
        INVARIANT: Must be pure and synchronous. No I/O, no network calls.
        The registry enforces this with a 2-second timeout wrapper.
        Strategy receives raw file content; returns classification.
        """

    @abstractmethod
    def build_generation_brief(
        self,
        classification: SourceClassification,
        user_context:   str,
        file_path:      str,
        test_plan:      List[Dict],
    ) -> str:
        """
        Returns the fully assembled prompt brief for test generation.
        Strategy owns: spec rules, Given/When/Then templates,
        naming constraints, lane requirements.
        user_context carries behavioral contracts — strategy enforces structure.
        MCP passes this brief to the agent. MCP never builds prompts itself.
        """

    @abstractmethod
    def validate_generated_test(
        self,
        test_content:         str,
        source_classification: SourceClassification,
    ) -> Dict:
        """
        Returns:
        {
          "valid": bool,
          "violations": [
            {"rule": str, "severity": "error"|"warning", "line": int}
          ],
          "spec_completeness": float,   # 0.0–1.0
          "lanes_covered": [TestLaneType],
          "missing_lanes":  [TestLaneType]
        }
        """
⚠
Spec Problem Tests generated from implementation alone mirror implementation bugs. build_generation_brief() must inject behavioral contracts — the user_context parameter is not optional. Strategy enforces spec structure; user supplies expected behavior.

class StrategyRegistry:
    def __init__(self, manifest_path: str = "strategies.yaml"):
        self._manifest    = self._load_manifest(manifest_path)
        self._strategies: dict[str, BaseStrategy] = {}
        self._active:     Optional[BaseStrategy] = None

    def load(self, strategy_id: str) -> BaseStrategy:
        entry = self._find_entry(strategy_id)
        strategy = self._import_strategy(entry)
        self._validate_contract(strategy)        # raises StrategyLoadError
        self._validate_classify_is_pure(strategy) # 2-second timeout check
        self._strategies[strategy_id] = strategy
        self._active = strategy
        return strategy

    def active(self) -> BaseStrategy:
        if not self._active:
            raise StrategyLoadError("No strategy loaded. Call load_strategy first.")
        return self._active

    def _validate_contract(self, strategy: BaseStrategy):
        # abc enforces abstract methods at instantiation time
        # additional runtime checks here for property types
        assert isinstance(strategy.id,              str), "id must be str"
        assert isinstance(strategy.target_language, str), "target_language must be str"
        lanes = strategy.get_test_lanes()
        assert len(lanes) > 0, "Strategy must declare at least one test lane"
        assert any(l.required for l in lanes), "At least one lane must be required"

# strategy/plugins/python_pytest_v1.py  (abridged)
import ast, re
from ..base import BaseStrategy, TestLane, TestLaneType, NamingConvention, BehavioralSpec, SourceClassification


class PythonPytestStrategy(BaseStrategy):

    @property
    def id(self) -> str: return "python_pytest_v1"

    @property
    def name(self) -> str: return "Python Pytest Strategy"

    @property
    def version(self) -> str: return "1.0.0"

    @property
    def target_language(self) -> str: return "python"

    @property
    def target_framework(self) -> str: return "pytest"

    def get_test_lanes(self) -> list:
        return [
            TestLane(TestLaneType.UNIT,        required=True,  file_glob="tests/unit/**/test_*.py",        coverage_threshold=0.80),
            TestLane(TestLaneType.INTEGRATION, required=True,  file_glob="tests/integration/**/test_*.py", coverage_threshold=0.60),
            TestLane(TestLaneType.CONTRACT,    required=False, file_glob="tests/contract/**/test_*.py",    coverage_threshold=0.0),
        ]

    def get_naming_conventions(self) -> NamingConvention:
        return NamingConvention(
            test_file_pattern=r"test_(?P<source>.+)\.py",
            test_method_pattern=r"test_(?P<action>.+)_when_(?P<condition>.+)_expects_(?P<expected>.+)",
        )

    def get_behavioral_specs(self) -> list:
        return [
            BehavioralSpec("service",    required_spec_fields=["given", "when", "then", "expected_state"], required_assertions_min=2),
            BehavioralSpec("repository", required_spec_fields=["given_db_state", "when", "then_db_state"],  required_assertions_min=1),
            BehavioralSpec("utility",    required_spec_fields=["input", "expected_output"],                  required_assertions_min=1),
        ]

    def classify_source_file(self, file_path: str, content: str) -> SourceClassification:
        # Pure AST-based classification — no I/O
        tree = ast.parse(content)
        metrics = self._extract_metrics(tree)
        class_type = self._infer_class_type(file_path, tree)
        return SourceClassification(
            class_type=class_type,
            complexity_score=min(10, metrics["avg_complexity"]),
            testability_issues=self._find_testability_issues(tree),
            required_lanes=self._lanes_for_type(class_type),
            ast_metrics=metrics,
        )

Enforcement is Advisory by Default in MCP MCP servers cannot technically force a calling agent to block on passed:false. Enforcement requires the agent's system prompt to declare the enforce tool response as a hard gate. This must be part of the agent configuration, not an assumption in the MCP.

09Repo Analysis Flow

Analysis is deterministic and LLM-free. It uses pathlib for discovery and Python's built-in ast module for parsing. The strategy plugin owns classification — the core only provides raw content and receives a SourceClassification.

cyclomatic_complexity Count If | For | While | Try | ExceptHandler nodes Complexity score, lane depth
has_side_effects Detect global state mutations, file I/O calls, network imports Testability issue flag
public_interface Functions/methods not prefixed with _ Spec scope — what needs coverage
decorator_patterns Extract ast.Decorator names Framework detection (pytest fixtures, FastAPI routes)

11Validation Flow

After the agent generates and writes a test file, validate_tests runs the generated content through the strategy's validate_generated_test() method, which checks rules the strategy declares. The MCP core adds structural checks that apply to any strategy.

Validation Layers

LayerOwnerWhat is CheckedOn Failure
Syntax parse MCP Core File is valid Python (AST parse without error) error
Naming conventions Strategy File name matches test_file_pattern; method names match test_method_pattern warning
Spec completeness Strategy Each test has all required Given/When/Then docstring fields for its class type error
Assertion count Strategy Test method has ≥ required_assertions_min assert statements error
Lane coverage Strategy Test file covers lanes declared as required for its source class type warning
No bare asserts MCP Core No assert True or assert x is not None as the only assertion warning
No skipped tests MCP Core No @pytest.mark.skip unless max_allowed_skips permits error

12Enforcement Gate

The enforce tool is the quality gate. It runs a full sweep and compares results against the thresholds declared by the loaded strategy. A passed:false response is a structured blocker — the agent system prompt should treat it as mandatory before proceeding.

13Reporting

Reports are built from in-memory session state via Pydantic schemas. V1 supports JSON output. Markdown rendering is a thin wrapper added post-V1.

# models/report.py
from pydantic import BaseModel
from datetime import datetime
from typing import List, Dict, Optional


class LaneCoverageReport(BaseModel):
    lane:      str
    required:  bool
    threshold: float
    actual:    float
    status:    str  # "OK" | "BREACH" | "NOT_MEASURED"
    files_missing: List[str]


class ViolationReport(BaseModel):
    file:     str
    rule:     str
    severity: str  # "error" | "warning"
    line:     int
    detail:   str


class QEReport(BaseModel):
    generated_at:     datetime
    strategy_id:      str
    strategy_version: str
    repo_path:        str
    enforce_passed:   Optional[bool]

    files_analysed:   int
    test_files_found: int
    coverage:         List[LaneCoverageReport]
    violations:       List[ViolationReport]
    test_plans:       List[Dict]   # all generated plans this session
    smells:           List[Dict]   # testability issues found in source
    blockers:         List[Dict]   # enforce blockers if run
    warnings:         List[Dict]

# core/logger.py
import logging, json, os
from datetime import datetime, timezone


class StructuredFormatter(logging.Formatter):
    def format(self, record: logging.LogRecord) -> str:
        entry = {
            "ts":      datetime.now(timezone.utc).isoformat(),
            "level":   record.levelname,
            "module":  record.module,
            "msg":     record.getMessage(),
        }
        if hasattr(record, "extra"):
            entry.update(record.extra)
        return json.dumps(entry)


def get_logger(name: str) -> logging.Logger:
    logger = logging.getLogger(name)
    handler = logging.StreamHandler()
    fmt = os.getenv("QE_MCP_LOG_FORMAT", "text")
    handler.setFormatter(
        StructuredFormatter() if fmt == "json"
        else logging.Formatter("%(asctime)s [%(levelname)s] %(module)s: %(message)s")
    )
    logger.addHandler(handler)
    logger.setLevel(os.getenv("QE_MCP_LOG_LEVEL", "INFO"))
    return logger

Adding a New Strategy (No Server Change Required)

Creating a new strategy plugin requires only two steps: implement BaseStrategy in a new Python file, and register it in strategies.yaml. The server picks it up at next start.

# strategy/plugins/karate_api_v1.py
from ..base import BaseStrategy, TestLane, TestLaneType, NamingConvention, BehavioralSpec, SourceClassification


class KarateApiStrategy(BaseStrategy):

    @property

Extending the Tool Surface Post-V1

New Tool Steps
  • Create tools/your_tool.py with a handler function
  • Register in tools/router.py
  • Declare tool schema in server.py tool registration
  • Add to models/report.py if it produces reportable output
Natural V2 Tools
  • analyse_dependencies: import graph, coupling score
  • diff_coverage: PR-scoped gap analysis (Blast Radius integration)
  • export_report: write report to file in JSON/Markdown/HTML
  • watch_repo: file-change triggered re-analysis
✓
Invariant: Strategy Contract is Stable The BaseStrategy ABC is versioned. Adding a new abstract method is a breaking change that requires a version bump and migration path. Any addition to the contract must be additive (optional with default) until the next major version, so existing plugins don't break silently.
