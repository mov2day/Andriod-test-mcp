QE-MCP

Quality Enforcement Model Context Protocol. Empowering AI agents to build, validate, and enforce robust testing architectures seamlessly.

What is QE-MCP?

QE-MCP acts as an intelligent bridge between AI coding agents and your codebase. Rather than letting AI guess how tests should be written, QE-MCP loads prescriptive strategy plugins (like python_pytest_v1 or android_compose_v1) that dictate exact rules for test lanes, naming conventions, and validation oracles.

The Agent Workflow

How an AI interacts with the repository using QE-MCP tools:

1

Analyze & Identify

The agent uses analyse_repo to scan the project. QE-MCP identifies source files, pairs them with existing tests, detects testability smells, and highlights coverage gaps across different test lanes.

2

Plan

For missing coverage, the agent calls generate_test_plan. QE-MCP outputs a prescriptive TC-001..N plan detailing exactly what edge cases and happy paths must be tested.

3

Generate

Using get_generation_brief, the AI receives strict instructions on syntax, mocking frameworks, and oracle structure, allowing it to write perfectly compliant test code.

4

Validate & Enforce

The agent submits the code to validate_tests. A 7-layer check ensures no bare assertions or skipped tests exist. Finally, enforce acts as the ultimate quality gate.

Core Tool Reference

analyse_repo

Deep Scan

Discovers files and uses the active strategy to classify components (e.g., ViewModels vs. Services). Generates an AI-ready prompt for semantic analysis.

validate_tests

7-Layer Validation

Validates syntax, checks file and method naming conventions, ensures oracle completeness (Given/When/Then), and prevents vacuous assertions.

enforce

Quality Gates

The final hurdle. Calculates coverage ratios across required lanes. If thresholds are breached or structural violations exist, the gate fails.

V2 Extensions

diff_coverage

PR-Scoped Analysis

Utilizes git diff to isolate gap analysis strictly to files changed in the current working branch, enabling faster PR feedback loops.

analyse_dependencies

Coupling Detection

Generates an import graph and calculates coupling scores to identify highly entangled modules that are inherently harder to test.

watch_repo

Continuous Sync

Runs a background watchdog process. Modifying source files automatically marks the session stale, prompting the agent to refresh analysis.

Quick Start

# 1. Install dependencies
pip install mcp pydantic pyyaml watchdog

# 2. Start the MCP server over stdio
python server.py

# 3. Configure your AI agent to use the server.
# For example, in Claude Desktop or via custom clients, map the
# command to execute `python /path/to/server.py`.
